# Implementation Records

> Every "major implementation" on this repository produces a seven-step
> record: **Current State Assessment → Gap Analysis → Improvement Strategy
> → Implementation Plan → Implementation → Validation Report →
> Recommendations for Next Iteration.**

This directory stores those records. Together with `governance/ecosystem-audit/`
they form the auditable history of how the Research Digital Infrastructure
of the *Judicial Data & AI Governance Lab* was built and modified.

---

## Standard file name

```
YYYY-MM-DD_<slug>.md
```

Where `<slug>` names the artefact touched (e.g., `WP01-publication-infrastructure`,
`JDGF-v1-conceptual-model`, `SPJEJ-case-study-scaffold`).

*The record is written **before** the changes are committed and updated
after validation. It is never rewritten to hide a false start — a false
start is documented in the *Recommendations for Next Iteration* section.*
