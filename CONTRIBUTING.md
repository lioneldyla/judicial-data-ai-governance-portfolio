# Contributing Guide

**Judicial Data & AI Governance Research Portfolio**

> Version: 1.0 | Author: DJORO Yoboukoua Lionel Arnaud | Last updated: 2026-07-06

---

Thank you for your interest in contributing to the Judicial Data & AI Governance Research Portfolio. This repository is an academic research portfolio dedicated to advancing **Judicial Data Governance**, **Digital Justice**, **Responsible Artificial Intelligence**, and the **Digital Transformation of Justice Systems**.

This guide ensures that all contributions maintain high standards of **scientific quality**, **transparency**, **reproducibility**, and **ethical integrity**.

## Before You Contribute

Please ensure that:

- Your contribution aligns with the repository's research programme
- Existing documentation has been reviewed
- Similar work has not already been submitted
- Sources are properly cited
- Intellectual property rights are respected

> For major scientific contributions, open an Issue first to discuss the proposal.

---

## Branch Naming Convention

Use descriptive branch names:

```
feature/wp02
feature/jdgf-framework
feature/research-dataset
feature/documentation
bugfix/references
docs/readme-update
```

---

## Commit Message Convention

Follow clear and descriptive commit messages:

```
docs: improve README
docs: update WP01 references
feat: add JDGF methodology
feat: publish WP02 draft
fix: correct bibliography formatting
refactor: reorganize repository structure
```

---

## Pull Request Guidelines

Each Pull Request should:

- Clearly describe the objective
- Explain the scientific or technical contribution
- Reference related issues where applicable
- Include updated documentation if needed
- Preserve repository consistency

Large contributions should be divided into smaller, reviewable pull requests whenever possible.

---

## Documentation Standards

Documentation should:

- Be written in clear academic English
- Use Markdown
- Follow consistent terminology
- Include citations where appropriate
- Clearly distinguish facts, interpretation, and opinion

---

## Citation Policy

If your work builds upon materials in this repository:

- Cite the relevant Working Paper, framework, dataset, or software
- Use the repository's `CITATION.cff` metadata whenever applicable
- Prefer the Zenodo DOI once available

---

## Research Integrity

All contributions must respect:

- Academic integrity
- Responsible AI principles
- FAIR data principles where applicable
- Open Science practices
- Ethical use of data
- Intellectual property rights

Plagiarism, data fabrication, falsification, or misleading claims will not be accepted.

---

## Code Quality

Research software contributions should:

- Be documented
- Include meaningful comments when appropriate
- Preserve reproducibility
- Follow existing project structure
- Include tests whenever feasible
