## Security Commitment

The Judicial Data & AI Governance Research Portfolio is committed to maintaining a **secure**, **trustworthy**, and **responsible** research environment. Although this repository primarily contains research outputs, documentation, datasets, and research software, security remains an essential component of scientific integrity and responsible digital research.

---

## Scope

This policy applies to all materials contained in this repository, including:

- Working Papers
- Research Frameworks
- Documentation
- Datasets
- Research Software
- Scripts
- GitHub Actions (if enabled)
- GitHub Pages (if enabled)

---

## Reporting Security Issues

If you discover a security issue, **please do not disclose it publicly** before it has been reviewed.

Examples include:

- Exposure of confidential information
- Security vulnerabilities in research software
- Accidental publication of sensitive data
- Authentication or authorization weaknesses
- Malicious code or dependencies
- Repository integrity concerns

Please report security issues privately through GitHub's [security reporting features](https://github.com/lioneldyla/judicial-data-ai-governance-portfolio/security) when available, or contact the repository maintainer directly.

**Include:**

- A clear description of the issue
- Steps to reproduce, if applicable
- Potential impact
- Suggested mitigation, if known

---

## Responsible Disclosure

Researchers and contributors are expected to follow responsible disclosure principles:

- Report issues privately
- Allow reasonable time for investigation
- Avoid public disclosure until mitigation is available
- Refrain from exploiting identified vulnerabilities

Constructive and ethical reporting is appreciated.

---

## Data Protection

This repository is committed to responsible handling of research data. Contributors must ensure that:

- Personal data are not published without appropriate authorization
- Confidential information is removed or anonymized
- Restricted datasets are not uploaded
- Applicable legal and ethical requirements are respected

Where possible, datasets should follow FAIR principles while preserving privacy and confidentiality.

---

## Artificial Intelligence Security

Projects involving Artificial Intelligence should:

- Document models and methodologies
- Identify known limitations
- Minimize security risks
- Protect sensitive information
- Promote transparency and accountability

AI systems should **support** — not replace — human judgment in justice-related contexts.

---

## Software Dependencies

Where research software is included:

- Use actively maintained dependencies
- Keep libraries reasonably up to date
- Review external packages before adoption
- Remove unused dependencies

---

## Supported Versions

Security updates will normally be provided for the latest maintained version of the repository. Older versions may not receive corrections unless the issue is considered critical.

---

## Security Best Practices

Contributors are encouraged to:

- Enable Multi-Factor Authentication on GitHub
- Protect personal access tokens
- Never commit passwords, API keys, or secrets
- Review pull requests before merging
- Verify external contributions

---

## No Sensitive Judicial Data

This repository must **not** contain:

- Personally identifiable judicial records
- Confidential government information
- Restricted case files
- Non-public institutional documents
- Any data whose publication would violate applicable law or ethical obligations

Research datasets should be anonymized whenever necessary.
